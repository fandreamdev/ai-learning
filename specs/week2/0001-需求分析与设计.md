## 1. 项目概述

### 1.1 项目定位

SmartQuery AI 是一款连接"人"与"数据"的智能桥梁软件，提供**"专业SQL模式"**与**"自然语言对话模式"**双轨并行功能。

### 1.2 核心价值

| 价值点 | 说明 |
|-------|------|
| **降低门槛** | 非技术人员无需SQL即可获取数据 |
| **提升效率** | 技术人员快速预览、辅助编写复杂SQL |
| **数据民主化** | 打破数据孤岛，分钟级获取业务指标 |

### 1.3 目标用户

| 用户类型 | 角色描述 | 核心需求 |
|---------|---------|---------|
| 业务人员 | 产品、运营、分析师、管理层 | 零代码获取业务指标 |
| 技术人员 | DBA、后端开发、数据工程师 | SQL辅助编写、测试数据生成 |

### 1.4 核心功能矩阵

| 功能模块 | SQL模式 | 对话模式 | 说明 |
|---------|---------|----------|------|
| 数据库连接 | ✅ | ✅ | 支持MySQL/PostgreSQL/ClickHouse等 |
| 语法高亮 | ✅ | - | 标准IDE功能 |
| 自动补全 | ✅ | - | SQL关键词/表名/列名 |
| 自然语言转SQL | - | ✅ | Text-to-SQL，准确率90%+ |
| 执行计划分析 | ✅ | - | EXPLAIN 分析 |
| 自动图表生成 | ✅ | ✅ | 根据结果维度匹配最佳图表 |
| 权限控制 | ✅ | ✅ | RBAC角色权限 |

## 2. 功能需求

### 2.1 双模式系统

#### 2.1.1 SQL专业模式

**FR-SQL-001: 数据库连接管理**

- 支持添加/编辑/删除数据库连接
- 支持的数据库类型：
  - MySQL 8.0+
  - PostgreSQL 12+
  - ClickHouse 21+
  - SQLite（本地文件）
- 连接配置：主机、端口、用户名、密码、数据库名
- 连接测试功能
- 连接状态监控（心跳检测）

**FR-SQL-002: SQL编辑器**

- 语法高亮（关键字、字符串、数字、注释）
- 自动补全（表名、列名、关键字）
- SQL格式化
- 多标签页支持
- 快捷键支持（Ctrl+Enter执行等）

**FR-SQL-003: 查询执行**

- 执行选中的SQL或全部SQL
- 执行计划展示（EXPLAIN）
- 结果分页展示
- 导出CSV/JSON格式
- 执行历史记录

**FR-SQL-004: SQL优化建议**

- 基于执行计划的慢查询识别
- 索引建议
- SQL改写建议

#### 2.1.2 自然语言对话模式

**FR-NL-001: 对话界面**

- 类似ChatGPT的对话式交互
- 支持多轮对话上下文
- 对话历史管理
- 对话导出

**FR-NL-002: Text-to-SQL引擎**

- 自然语言理解
- SQL自动生成
- 多数据库方言适配
- 语义层集成

**FR-NL-003: 用户确认流程**

- 生成SQL预览展示
- SQL原理解释（"我为您生成了如下查询：按地区分组..."）
- 用户确认后执行
- 执行结果展示

**FR-NL-004: 上下文管理**

- 会话级上下文保持
- 跨会话上下文继承
- 上下文长度限制与优化

### 2.2 自动图表与分析

**FR-VIZ-002: 图表交互**

- 图表类型切换
- 数据筛选/钻取
- 缩放与平移
- 图表下载（PNG/SVG）

**FR-VIZ-003: 数据洞察**

- 异常值标注
- 趋势解读文字
- 一句话数据洞察生成

### 2.3 语义层/元数据管理

**FR-META-001: 表结构管理**

## 4. 安全设计

### 4.1 数据库安全

**SEC-001: 最小权限原则**

```
自然语言模式数据库账号权限:
GRANT SELECT ON target_db.* TO 'smartquery_nl'@'%';

禁止权限:
- INSERT
- UPDATE
- DELETE
- DROP
- TRUNCATE
- ALTER
- CREATE
- GRANT/REVOKE
```

**SEC-002: SQL AST分析**

在执行前使用AST分析器检测：

| 检测类型 | 拦截规则 |
|---------|---------|
| DML操作 | INSERT/UPDATE/DELETE |
| DDL操作 | CREATE/DROP/ALTER/TRUNCATE |
| 危险函数 | LOAD_FILE, INTO OUTFILE, BENCHMARK, SLEEP |
| 跨库查询 | 数据库名.表名（白名单外） |
| 注释注入 | /* */, -- 等 |

**SEC-003: 提示词注入防护**

- 敏感词过滤
- 越狱提示词检测
- 输入长度限制

### 4.2 数据安全

**SEC-004: 敏感数据脱敏**

| 字段类型 | 脱敏方式 | 示例 |
|---------|---------|------|
| 手机号 | 中间4位掩码 | 138****1234 |
| 身份证 | 前3后4位掩码 | 110***********1234 |
| 邮箱 | 域名前缀掩码 | a***@example.com |
| 银行卡 | 前6后4位掩码 | 622202******1234 |
| 薪资 | 范围模糊 | 15k-20k |

**SEC-005: 传输安全**

- HTTPS强制
- 请求签名验证
- API密钥管理

### 4.3 访问控制

**SEC-006: 认证**

- 用户名密码认证
- 邮箱验证码登录（可选）
- Session管理
- JWT令牌

**SEC-007: 授权**

- RBAC角色权限控制
- 资源级权限
- 操作级权限

### 5.2 核心模块职责

| 模块 | 职责 | 技术栈 |

| 图表服务 | 数据转换、图表推荐 | **Rust/Serde** |
| 用户服务 | 认证授权、用户管理 | **Rust/JWT** |

### 5.3 数据流设计

#### 5.3.1 SQL模式查询流程

```
用户输入SQL → 前端 → API网关 → SQL执行服务 → AST安全检测 → 数据库 → 结果返回 → 前端展示
```

#### 5.3.2 自然语言转SQL流程

```
用户提问 → 前端 → API网关 → NL转SQL服务
                                     ↓
                              Schema检索(RAG)
                                     ↓
                              LLM生成SQL
                                     ↓
                              SQL预览返回前端
                                     ↓
用户确认 → 执行 → 结果 → 图表生成 → 前端展示
```

## 6. 数据模型

### 6.1 核心实体

#### 6.1.1 用户 (User)

```rust
use serde::{Deserialize, Serialize};
use sqlx::FromRow;

#[derive(Debug, Clone, Serialize, Deserialize, FromRow)]
pub struct User {
    pub id: String,           // UUID
    pub username: String,      // 用户名
    pub email: String,         // 邮箱
    pub password_hash: String, // 密码哈希
    pub role: UserRole,        // 角色
    pub is_active: bool,
    pub created_at: chrono::DateTime<chrono::Utc>,
    pub updated_at: chrono::DateTime<chrono::Utc>,
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(rename_all = "lowercase")]
pub enum UserRole {
    Admin,           // 管理员
    Analyst,         // 数据分析师
    Developer,       // 开发人员
    Business,        // 业务人员
}

impl std::fmt::Display for UserRole {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            UserRole::Admin => write!(f, "admin"),
            UserRole::Analyst => write!(f, "analyst"),
            UserRole::Developer => write!(f, "developer"),
            UserRole::Business => write!(f, "business"),
        }
    }
}
```

#### 10.3.1 SQL AST 安全分析

```rust
use sqlparser::ast::{Statement, Visitor};
use sqlparser::dialect::MySqlDialect;
use sqlparser::parser::Parser;

#[derive(Debug)]
pub struct SqlAnalysisResult {
    pub is_safe: bool,
    pub blocked_reason: Option<String>,
    pub warnings: Vec<String>,
}

pub fn analyze_sql_safety(sql: &str) -> SqlAnalysisResult {
    let dialect = MySqlDialect {};
    let ast = match Parser::parse_sql(&dialect, sql) {
        Ok(ast) => ast,
        Err(_) => return SqlAnalysisResult {
            is_safe: false,
            blocked_reason: Some("SQL语法解析失败".to_string()),
            warnings: vec![],
        },
    };

    let mut blocked_reason = None;
    let mut warnings = vec![];

    for statement in &ast {
        match statement {
            Statement::Update(_) => {
                blocked_reason = Some("禁止 UPDATE 操作".to_string());
            }
            Statement::Delete(_) => {
                blocked_reason = Some("禁止 DELETE 操作".to_string());
            }
            Statement::Drop(_) => {
                blocked_reason = Some("禁止 DROP 操作".to_string());
            }
            Statement::Insert(_) => {
                blocked_reason = Some("禁止 INSERT 操作".to_string());
            }
            Statement::Truncate(_) => {
                blocked_reason = Some("禁止 TRUNCATE 操作".to_string());
            }
            Statement::Alter(_) => {
                blocked_reason = Some("禁止 ALTER 操作".to_string());
            }
            Statement::Grant(_) | Statement::Revoke(_) => {
                blocked_reason = Some("禁止权限变更操作".to_string());
            }
            _ => {}
        }

        // 检查危险函数
        let func_checker = FunctionChecker::new();
        func_checker.visit_statement(statement);
        warnings.extend(func_checker.warnings);
    }

    SqlAnalysisResult {
        is_safe: blocked_reason.is_none(),
        blocked_reason,
        warnings,
    }
}

struct FunctionChecker {
    dangerous_functions: Vec<&'static str>,
    warnings: Vec<String>,
}

impl FunctionChecker {
    fn new() -> Self {
        Self {
            dangerous_functions: vec![
                "LOAD_FILE",
                "INTO OUTFILE",
                "BENCHMARK",
                "SLEEP",
                "SHUTDOWN",
            ],
            warnings: vec![],
        }
    }
}
```

#### 10.3.2 API 端点实现

```rust
use axum::{
    extract::{Path, State, Query},
    http::StatusCode,
    response::Json,
    routing::{post, get},
    Router,
};
use serde::{Deserialize, Serialize};
use sqlx::PgPool;

// 请求类型
#[derive(Debug, Deserialize)]
pub struct SqlExecuteRequest {
    pub connection_id: String,
    pub sql: String,
    pub timeout: Option<u64>,
}

// 响应类型
#[derive(Debug, Serialize)]
pub struct SqlExecuteResponse {
    pub query_id: String,
    pub columns: Vec<String>,
    pub rows: Vec<Vec<serde_json::Value>>,
    pub row_count: i64,
    pub duration_ms: i64,
}

// 应用状态
pub struct AppState {
    pub db: PgPool,
    pub llm_client: LlmClient,
}

// SQL执行端点
async fn execute_sql(
    State(state): State<AppState>,
    Json(req): Json<SqlExecuteRequest>,
) -> Result<Json<SqlExecuteResponse>, StatusCode> {
    // 1. SQL安全分析
    let analysis = analyze_sql_safety(&req.sql);
    if !analysis.is_safe {
        return Err(StatusCode::FORBIDDEN);
    }

    // 2. 执行查询
    let start = std::time::Instant::now();
    let result = state.db.query(&req.sql).await.map_err(|e| {
        tracing::error!("SQL执行失败: {}", e);
        StatusCode::INTERNAL_SERVER_ERROR
    })?;

    let duration_ms = start.elapsed().as_millis() as i64;

    Ok(Json(SqlExecuteResponse {
        query_id: uuid::Uuid::new_v4().to_string(),
        columns: result.columns,
        rows: result.rows,
        row_count: result.row_count as i64,
        duration_ms,
    }))
}

// NL转SQL端点
async fn nl_to_sql(
    State(state): State<AppState>,
    Json(req): Json<NlConvertRequest>,
) -> Result<Json<NlConvertResponse>, StatusCode> {
    // 1. Schema检索
    let schema = state.schema_service.retrieve(&req.question).await
        .map_err(|_| StatusCode::INTERNAL_SERVER_ERROR)?;

    // 2. LLM生成SQL
    let response = state.llm_client.generate_sql(&schema, &req.question).await
        .map_err(|_| StatusCode::INTERNAL_SERVER_ERROR)?;

    Ok(Json(NlConvertResponse {
        sql: response.sql,
        explanation: response.explanation,
        confidence: response.confidence,
    }))
}
```
