## 1. 项目概述

ProjectAlpha 是一个轻量级的项目管理工具，核心功能是对 **Ticket**（任务/问题/需求/待办事项）进行创建、分类、跟踪和管理。系统通过标签、状态、优先级等维度组织 Ticket，帮助中小型团队高效管理项目工作内容。

### 1.1 术语定义

| 术语              | 说明                                                       |
| ----------------- | ---------------------------------------------------------- |
| Ticket            | 系统中的基本工作单元，可表示任务、问题、需求或待办事项     |
| 标签 (Tag)        | 用于对 Ticket 进行分类的文字标记，如 `bug`、`feat`、`perf` |
| 状态 (Status)     | Ticket 的当前处理阶段：待处理、处理中、已完成、已关闭      |
| 优先级 (Priority) | Ticket 的紧急程度：低、中、高、紧急                        |
| 负责人 (Assignee) | 被指派处理该 Ticket 的人员                                 |

### 1.2 用户角色

本期仅支持单一角色——**普通用户**，拥有所有 Ticket 的完整操作权限（CRUD），不区分管理员与普通成员。

## 2. 项目目标

| 编号 | 目标                     | 衡量标准                                       |
| ---- | ------------------------ | ---------------------------------------------- |
| G1   | 统一管理任务、问题和需求 | 所有类型的工作项均可通过 Ticket 表达和管理     |
| G2   | 通过标签实现灵活分类     | 用户可自由输入标签，不受预定义限制             |
| G3   | 通过状态跟踪处理进度     | 支持 4 种状态流转，状态变更即时生效            |
| G4   | 提供清晰的前后端分层架构 | 前后端完全分离，通过 REST API 通信             |
| G5   | 便于后续扩展             | 架构预留权限、评论、附件、通知等模块的扩展空间 |

## 3. 功能需求

### 3.1 Ticket 管理（核心模块）

| 编号 | 功能             | 说明                                    | 验收标准                                             |
| ---- | ---------------- | --------------------------------------- | ---------------------------------------------------- |
| F01  | 创建 Ticket      | 填写标题、描述、优先级、标签、负责人    | 必填字段为空时提示错误；创建成功后列表自动刷新       |
| F02  | 查看 Ticket 列表 | 以列表形式展示所有 Ticket，支持分页     | 默认按创建时间降序；空列表显示空状态提示             |
| F03  | 查看 Ticket 详情 | 点击进入详情页，展示完整信息            | 展示所有字段信息，包含格式化的时间戳                 |
| F04  | 编辑 Ticket      | 修改标题、描述、优先级、标签、负责人    | 编辑后 updated_at 自动更新；修改即时生效             |
| F05  | 删除 Ticket      | 删除指定 Ticket                         | 删除前弹出二次确认弹窗；删除后列表自动刷新           |
| F06  | 修改状态         | 在 待处理/处理中/已完成/已关闭 之间切换 | 支持在详情页和列表页快捷切换状态                     |
| F07  | 设置优先级       | 设置为 低/中/高/紧急                    | 优先级以不同颜色标识（低-灰、中-蓝、高-橙、紧急-红） |
| F08  | 添加/移除标签    | 为 Ticket 绑定或解除标签                | 支持在编辑时输入新标签或选择已有标签                 |
| F09  | 指定负责人       | 为 Ticket 分配负责人                    | 负责人为自由文本输入                                 |
| F10  | 时间记录         | 自动记录创建时间与更新时间              | 时间格式统一为 `YYYY-MM-DD HH:mm:ss`                 |

### 3.2 分类与筛选

| 编号 | 功能         | 说明                              | 验收标准                                         |
| ---- | ------------ | --------------------------------- | ------------------------------------------------ |
| F11  | 按标签筛选   | 选择一个或多个标签进行过滤        | 多标签为"或"关系；清除后恢复全部                 |
| F12  | 按状态筛选   | 复选框多选状态进行过滤            | 支持同时查看多个状态的 Ticket                    |
| F13  | 按优先级筛选 | 复选框多选优先级进行过滤          | 支持同时选择多个优先级                           |
| F14  | 按负责人筛选 | 下拉选择负责人进行过滤            | 下拉列表自动聚合已有的负责人                     |
| F15  | 关键字搜索   | 对标题和描述进行模糊匹配          | 支持输入防抖（300ms），最少输入 2 个字符触发搜索 |
| F16  | 排序         | 按创建时间或更新时间升序/降序排列 | 默认按创建时间降序；切换排序时保持筛选条件       |

### 3.3 状态管理

系统预设以下 Ticket 状态，不支持自定义：

```
                    +--------+
                    | 待处理 |
                    +---+----+
                        |
                        v
                    +--------+
               +--->| 处理中 |---+
               |    +---+----+   |
               |        |        |
               |        v        v
               |    +--------+  +--------+
               |    | 已完成 |->| 已关闭 |
               |    +--------+  +--------+
               |                    ^
               +--------------------+
```

**状态流转规则：**

| 当前状态 | 可流转至           |
| -------- | ------------------ |
| 待处理   | 处理中、已关闭     |
| 处理中   | 已完成、已关闭     |
| 已完成   | 已关闭             |
| 已关闭   | 待处理（重新打开） |

### 3.4 标签说明

标签不提供独立管理功能（无独立的增删改页面）。标签在创建/编辑 Ticket 时直接输入，系统自动收集已有标签供选择。

**标签规则：**

- 标签长度：1 ~ 20 个字符
- 标签格式：允许中英文、数字和连字符 `-`
- 单个 Ticket 最多关联 10 个标签
- 标签不区分大小写，存储时统一转为小写

## 4. 数据模型设计

### 4.1 Ticket 实体

| 字段        | 类型                                       | 必填 | 默认值                      | 说明                        |
| ----------- | ------------------------------------------ | ---- | --------------------------- | --------------------------- |
| id          | INT UNSIGNED                               | 是   | AUTO_INCREMENT              | 主键，自增                  |
| title       | VARCHAR(200)                               | 是   | -                           | Ticket 标题，不可为空字符串 |
| description | TEXT                                       | 否   | NULL                        | Ticket 详细描述             |
| status      | ENUM('open','in_progress','done','closed') | 是   | 'open'                      | 当前状态                    |
| priority    | ENUM('low','medium','high','urgent')       | 是   | 'medium'                    | 优先级                      |
| assignee    | VARCHAR(100)                               | 否   | NULL                        | 负责人名称                  |
| tags        | JSON                                       | 否   | '[]'                        | 标签列表，JSON 数组格式     |
| created_at  | DATETIME                                   | 是   | CURRENT_TIMESTAMP           | 创建时间                    |
| updated_at  | DATETIME                                   | 是   | CURRENT_TIMESTAMP ON UPDATE | 更新时间                    |

### 4.2 枚举值映射（前后端统一）

**状态 (status)：**

| 存储值      | 中文显示 |
| ----------- | -------- |
| open        | 待处理   |
| in_progress | 处理中   |
| done        | 已完成   |
| closed      | 已关闭   |

**优先级 (priority)：**

| 存储值 | 中文显示 | 显示颜色       |
| ------ | -------- | -------------- |
| low    | 低       | 灰色 `#999999` |
| medium | 中       | 蓝色 `#1890FF` |
| high   | 高       | 橙色 `#FA8C16` |
| urgent | 紧急     | 红色 `#F5222D` |

## 5. API 接口设计

### 5.1 通用约定

- **Base URL**: `/api/v1`
- **Content-Type**: `application/json`
- **时间格式**: ISO 8601（`2026-05-17T09:15:00Z`）
- **分页参数**: `page`（从 1 开始）、`page_size`（默认 20，最大 100）

**通用响应结构：**

```json
// 成功（单条数据）
{
  "code": 0,
  "message": "success",
  "data": { ... }
}

// 成功（列表数据）
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [ ... ],
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}

// 失败
{
  "code": 40001,
  "message": "标题不能为空",
  "data": null
}
```

### 5.2 Ticket 接口

#### 5.2.1 获取 Ticket 列表

```
GET /api/v1/tickets
```

**查询参数：**

| 参数       | 类型    | 必填 | 默认值     | 说明                                           |
| ---------- | ------- | ---- | ---------- | ---------------------------------------------- |
| status     | String  | 否   | -          | 按状态筛选，多个用逗号分隔：`open,in_progress` |
| priority   | String  | 否   | -          | 按优先级筛选，多个用逗号分隔：`high,urgent`    |
| assignee   | String  | 否   | -          | 按负责人筛选，精确匹配                         |
| tag        | String  | 否   | -          | 按标签筛选                                     |
| keyword    | String  | 否   | -          | 对标题和描述模糊搜索                           |
| sort_by    | String  | 否   | created_at | 排序字段：`created_at` / `updated_at`          |
| sort_order | String  | 否   | desc       | 排序方向：`asc` / `desc`                       |
| page       | Integer | 否   | 1          | 页码，从 1 开始                                |
| page_size  | Integer | 否   | 20         | 每页条数，最大 100                             |

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [
      {
        "id": 1,
        "title": "修复登录页面验证码不刷新",
        "status": "in_progress",
        "priority": "high",
        "assignee": "张三",
        "tags": ["bug", "frontend"],
        "created_at": "2026-05-10T14:30:00Z",
        "updated_at": "2026-05-17T09:15:00Z"
      }
    ],
    "total": 42,
    "page": 1,
    "page_size": 20
  }
}
```

#### 5.2.2 获取 Ticket 详情

```
GET /api/v1/tickets/:id
```

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "title": "修复登录页面验证码不刷新",
    "description": "用户反馈在登录页面点击验证码图片后...",
    "status": "in_progress",
    "priority": "high",
    "assignee": "张三",
    "tags": ["bug", "frontend"],
    "created_at": "2026-05-10T14:30:00Z",
    "updated_at": "2026-05-17T09:15:00Z"
  }
}
```

#### 5.2.3 创建 Ticket

```
POST /api/v1/tickets
```

**请求体：**

```json
{
  "title": "修复登录页面验证码不刷新",
  "description": "用户反馈在登录页面点击验证码图片后...",
  "priority": "high",
  "assignee": "张三",
  "tags": ["bug", "frontend"]
}
```

**校验规则：**

| 字段        | 校验                                  |
| ----------- | ------------------------------------- |
| title       | 必填，长度 1~200                      |
| description | 选填，无长度限制                      |
| priority    | 选填，必须为合法枚举值，默认 `medium` |
| assignee    | 选填，长度 1~100                      |
| tags        | 选填，数组最多 10 项，每项 1~20 字符  |

#### 5.2.4 更新 Ticket

```
PUT /api/v1/tickets/:id
```

**请求体：** 同创建，所有字段均为选填（仅传需要修改的字段）。

#### 5.2.5 更新 Ticket 状态

```
PATCH /api/v1/tickets/:id/status
```

**请求体：**

```json
{
  "status": "in_progress"
}
```

> 后端需校验状态流转是否合法（参见 3.3 节状态流转规则）。

#### 5.2.6 删除 Ticket

```
DELETE /api/v1/tickets/:id
```

> 返回 `204 No Content` 或 `{ "code": 0, "message": "success" }`。

### 5.3 辅助接口

#### 5.3.1 获取所有标签

```
GET /api/v1/tags
```

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": ["bug", "feat", "frontend", "perf"]
}
```

#### 5.3.2 获取所有负责人

```
GET /api/v1/assignees
```

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": ["张三", "李四", "王五"]
}
```

## 6. 前端页面设计

### 6.1 整体布局

页面分为上下两大区域：**Header（顶部栏）** 和 **Body（主体区域）**。
主体区域左右分栏：左侧为 **Sidebar（筛选栏）**，右侧为 **TicketList（内容列表）**。

```
+----------------------------------------------------------------------+
|  [Logo] ProjectAlpha       [搜索关键字......]       [+ 新建 Ticket]   |
+----------------------------------------------------------------------+
|              |                                                        |
|  Sidebar     |  Ticket 列表                          排序: 创建时间v  |
|              |  +------+----------+------+------+--------+----------+ |
|  状态        |  | ID   | 标题     | 状态 | 优先级| 负责人 | 更新时间 | |
|  ----------  |  +------+----------+------+------+--------+----------+ |
|  [ ] 待处理  |  | #001 | 修复登录 | 处理中| 高   | 张三   | 05-17   | |
|  [ ] 处理中  |  | #002 | 新增导出 | 待处理| 中   | 李四   | 05-16   | |
|  [x] 已完成  |  | #003 | 优化查询 | 已完成| 低   | 王五   | 05-15   | |
|  [ ] 已关闭  |  | #004 | 接口重构 | 处理中| 高   | 赵六   | 05-14   | |
|              |  +------+----------+------+------+--------+----------+ |
|  优先级      |  |                  < 1  2  3  4  5 >                 | |
|  ----------  |  +----------------------------------------------------+ |
|  [ ] 低      |                                                        |
|  [x] 中      |                                                        |
|  [ ] 高      |                                                        |

|              |                                                        |
|  负责人      |                                                        |
```

**组件说明：**

| 区域       | 组件            | 说明                                      |

| Header     | SearchBar       | 关键字搜索框，300ms 防抖，最少 2 字符触发 |
| Header     | NewTicketButton | 新建 Ticket 按钮，点击弹出创建表单        |
| Sidebar    | 状态筛选        | 复选框，可多选，勾选后即时过滤            |
| Sidebar    | 优先级筛选      | 复选框，可多选，勾选后即时过滤            |
| Sidebar    | 负责人筛选      | 下拉选择框，数据来自 `/api/v1/assignees`  |
| Sidebar    | 标签筛选        | 复选框，数据来自 `/api/v1/tags`，可多选   |
| TicketList | 列表表格        | 展示 Ticket 核心信息，点击行进入详情      |
| TicketList | 排序控件        | 按创建时间/更新时间排序                   |
| TicketList | 分页控件        | 底部分页导航，显示总条数                  |

### 6.3 新建/编辑 Ticket 表单（弹窗）

```
+--------------------------------------------------+
|           新建 Ticket                      [ X ]  |
+--------------------------------------------------+
|                                                    |
|  标题 *:  [________________________________]       |
|                                                    |
|  描述:    [________________________________]       |
|           [________________________________]       |
|           [________________________________]       |
|                                                    |
|  优先级:  ( ) 低  (*) 中  ( ) 高  ( ) 紧急        |
|                                                    |
|  状态:    [待处理           v]                      |
|                                                    |
|  负责人:  [________________ v]                      |
|                                                    |
|  标签:    [bug] [feat] [+ 添加标签]                 |
|                                                    |
|              [取消]          [保存]                 |
+--------------------------------------------------+
```

**交互细节：**

| 操作               | 行为                                                   |
| ------------------ | ------------------------------------------------------ |
| 点击 [X] 或 [取消] | 若表单已修改，弹出"放弃修改？"确认；未修改直接关闭     |
| 点击 [保存]        | 前端校验 → 提交 API → 成功后关闭弹窗并刷新列表         |
| 标题为空提交       | 输入框标红，显示"标题不能为空"                         |
| [+ 添加标签]       | 弹出文本输入框，输入后回车添加；下方展示已有标签可点选 |
| 编辑模式           | 表单标题变为"编辑 Ticket #xxx"，预填当前数据           |

### 6.6 加载与异常状态

| 场景                 | 表现                                          |

## 10. 数据库设计

### 10.1 DDL 建表语句

```sql
CREATE DATABASE IF NOT EXISTS project_alpha
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

USE project_alpha;

CREATE TABLE tickets (
  id          INT UNSIGNED    NOT NULL AUTO_INCREMENT PRIMARY KEY,
  title       VARCHAR(200)    NOT NULL                COMMENT 'Ticket 标题',
  description TEXT            NULL                    COMMENT '详细描述',
  status      ENUM('open','in_progress','done','closed')
                              NOT NULL DEFAULT 'open' COMMENT '状态',
  priority    ENUM('low','medium','high','urgent')
                              NOT NULL DEFAULT 'medium' COMMENT '优先级',
  assignee    VARCHAR(100)    NULL                    COMMENT '负责人',
  tags        JSON            NULL DEFAULT ('[]')     COMMENT '标签列表',
  created_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP
                              ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

  INDEX idx_status     (status),
  INDEX idx_priority   (priority),
  INDEX idx_assignee   (assignee),
  INDEX idx_created_at (created_at),
  INDEX idx_updated_at (updated_at),
  FULLTEXT INDEX ft_title_desc (title, description)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='Ticket 工单表';
```

### 10.2 索引说明

| 索引           | 类型     | 说明               |
| -------------- | -------- | ------------------ |
| PRIMARY (id)   | 主键     | 唯一标识           |
| idx_status     | 普通索引 | 加速按状态筛选     |
| idx_priority   | 普通索引 | 加速按优先级筛选   |
| idx_assignee   | 普通索引 | 加速按负责人筛选   |
| idx_created_at | 普通索引 | 加速按创建时间排序 |
| idx_updated_at | 普通索引 | 加速按更新时间排序 |
| ft_title_desc  | 全文索引 | 加速关键字模糊搜索 |

### 10.3 种子数据

项目应提供至少 **30 条** 种子数据，覆盖以下场景：

- 四种状态各至少 5 条
- 四种优先级各至少 5 条
- 至少包含 5 种不同的标签
- 至少包含 4 个不同的负责人
- 时间跨度覆盖最近 30 天

### 15.2 Mock 方案

```typescript
import { http, HttpResponse } from 'msw'

export const handlers = [
  http.get('/api/v1/tickets', () => {
    return HttpResponse.json({
      code: 0,
      message: 'success',
      data: {
        items: [
          { id: 1, title: '测试 Ticket', status: 'open', priority: 'medium', ... },
        ],
        total: 1,
        page: 1,
        page_size: 20
      }
    })
  }),
]
```

### 15.3 跨域处理

开发环境通过 Vite 代理解决跨域：

```typescript
// frontend/vite.config.ts
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
    },
  },
})
```

> 后端 FastAPI 在生产环境通过 Nginx 反向代理后无需 CORS；开发环境可在 `app/middlewares/cors.py` 中开启 `CORSMiddleware` 允许前端开发服务器（`http://localhost:5173`）跨域访问。

### 15.4 联调检查清单

| 序号 | 检查项        | 说明                                  |

```yaml
      - mysql_data:/var/lib/mysql
      - ./database/init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost']
      interval: 10s
      timeout: 5s
      retries: 5

  backend:
    build: ./backend
    container_name: pa-backend
    environment:
      DB_HOST: mysql
      DB_PORT: 3306
      DB_USER: root
      DB_PASSWORD: ${DB_PASSWORD}
      DB_NAME: project_alpha
      APP_ENV: production
      APP_PORT: 8000
      DATABASE_URL: mysql+pymysql://root:${DB_PASSWORD}@mysql:3306/project_alpha
    ports:
      - '8000:8000'
    depends_on:
      mysql:
        condition: service_healthy
    # 容器内启动命令：先执行 alembic 迁移，再启动 uvicorn
    command: >

    build: ./frontend
```

```dockerfile
FROM python:3.13-slim

RUN poetry config virtualenvs.create false \
    && poetry install --no-interaction --no-ansi --without dev

# 复制应用代码
```
