## 1. 验收结论

实际后端主入口为 `week2/backend/src/main.rs` 挂载 `/api/v1`，实际接口集中在 `week2/backend/src/api/routes.rs`。备用模块如 `api/auth.rs`、`api/conversations.rs` 等不在主路由中，本次只作为参考，不作为实际接口完成依据。

总体结论：后端核心 API 基本接入完成，但不完全覆盖需求和实现计划。本轮已修复前端认证响应解包、token 使用、SQL/NL 原生 fetch 鉴权、图表 hook 请求体、NL 确认执行链路、OpenAI 兼容响应解析、ClickHouse 最小真实链路、PNG data URL 导出、备用模拟返回、审计日志查询、语义定义管理、SQL 工作区 Schema 展示、CSV/JSON 导出和基础 EXPLAIN 优化建议。剩余主要缺口是数据权限不完整、对话历史前端未接入、对话导出缺失，以及部分高级 SQL/图表能力未完全产品化。

## 6. 项目整体架构

### 6.1 后端架构

```text
HTTP /api/v1

  -> Services
       - Auth/JWT/Password

       - UserRepo

       - ConversationRepo
       - MetricRepo
       - AuditRepo
  -> Metadata DB (PostgreSQL via SQLx)
  -> Target DB Pools (PostgreSQL/MySQL/SQLite)
  -> External LLM (OpenAI-compatible chat completions)
```

### 6.3 实际数据流

- SQL 执行：前端 SQL 页 -> `/api/v1/sql/execute` -> JWT/RBAC -> 连接配置 -> 目标数据库连接池 -> SQL 安全分析 -> SQLx 查询 -> 脱敏 -> 查询历史/审计 -> 前端表格展示。
- NL 转 SQL：前端对话页 -> `/api/v1/nl/convert` -> JWT/RBAC -> 连接配置 -> schema 获取 -> 可选对话上下文 -> LLM -> SQL 安全分析 -> 前端展示 SQL。
- 图表生成：前端 hooks/预期结果区 -> `/api/v1/charts/*` -> JWT/RBAC -> `ChartGenerator` -> ECharts 配置或导出文件。
- 指标执行：客户端 -> `/api/v1/metrics/{id}/execute` -> 指标定义 -> 维度替换 -> SQL 安全分析 -> 目标数据库执行 -> 格式化返回。
